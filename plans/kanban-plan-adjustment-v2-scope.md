# Kanban Plan Adjustment V2 Scope

## Purpose

Define the scope for a second iteration where user feedback can drive plan revisions before implementation starts.

## In Scope

- Allow users to send change requests to the planning agent from the Kanban UI.
- Keep the task in approval workflow until the revised plan is reviewed again.
- Support multiple rounds of feedback and revised planning output for the same task.
- Preserve a clear audit trail of user feedback and planner revisions in task output.
- Expose the revision status in the board and graph experiences.

## Out of Scope

- Detailed API schemas
- Detailed prompt templates
- Detailed UI layout and interaction specs
- Migration and rollout sequencing
- Test matrix and implementation task breakdown

## Deliverable

This document captures scope only. Detailed design and implementation planning will be written separately.
